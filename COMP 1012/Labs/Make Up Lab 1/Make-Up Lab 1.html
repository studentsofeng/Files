QUESTIONS ON SIMPLE EXPRESSIONS

What is printed by print (4**3 / 3 // 2) ? 10. Correct!

In this section the questions involve small integers (and occasional floats) and the basic math operations: +, -, *, /, //, % and **. There are no brackets, so you need to know the precedence with which these operators are evaluated. In brief, there are three rules, as covered by the acronym BEDMAS (meaning, Brackets, Exponentiation, Division and Multiplication, Addition and Subtraction):

  1. Evaluate powers (exponentiation) first, right to left.
    That is, 2**3**4 is the same as 2**(3**4), which is the huge number 2417851639229258349412352, NOT (2**3)**4, which yields 8**4 = 4096.
  2. Evaluate multiplications and division next, left to right.
    So 1 + 2 * 3 // 4 % 7 is the same as 1 + (((2 * 3) // 4) % 7), with the addition done last.
  3. Evaluate additions and subtractions next, left to right.
    The expression 1 - 2 * 4 - 2**3 / 2 is done as follows:
    • First the power, leaving 1 - 2 * 4 - 8 / 2.
    • Then the multiplications and divisions, left to right, leaving 1 - 8 - 4.0.
    • Then the additions and subtractions left to right, giving (1 - 8) - 4.0 = -11.0.

One additional consideration with the division operators is the possibility of dividing by 0. Python has a rule: you cannot follow /, // or % by 0. If that occurs, the answer will be an error, and you should enter Error as your response.

If the expression involves a float constant (like 5.), then the final result of the expression will be float as well. Enter the result with a decimal point. With regard to division, remember:

  • that the // operator always produces an integer value, but if one of the operands is float, the result will be of type float.
  • that the / operator always produces a result of type float, even if both of the operands are of type int.

QUESTIONS ON BOOLEAN EXPRESSIONS

What is printed by print (1 == 1 and 2 or 3) ? 2 Correct!

Python is unusual among computer languages in how it mixes relational operators ( <, ≤, ==, !=, > ≥) and logical operators (and and or). But once again there are a few rules.

  1. Evaluate all the mathematical expressions first, as described above. The expression 5 + 3 == 2 * 4 > 3 is evaluated like (5 + 3) == (2 * 4) > 3.
  2. Evaluate all comparisons left to right, and combine them with and. So the expression above becomes 8 == 8 > 3, which is evaluated as (8 == 8) and (8 > 3). Since both comparisons yield True, the result is True. Few other languages evaluate a sequence of comparisons this way. Note that if the expression were 5 + 3 < 2 * 4 > 3 instead, it would simplify to 8 < 8 > 3 and then to (8 < 8) and (8 > 3). In this case, the leftmost comparison 8 < 8 is False, and we know right away it would make the chain of comparisons False, and so Python would not actually evaluate 8 > 3 as its value is irrelevant. Sometimes not evaluating code in an expression has side-effects (e.g., a function does not get called to print out a value). That is why it is important to know that evaluation is left to right.
  3. Next evaluate ands left to right.
    You probably know the rule for evaluating xx and yy when xx and yy are both bool: the result is True if and only if both xx and yy are True. But what is the value of (5 and 3)? Python has a more general definition of and than most languages. First, the result of xx and yy is going to be one of the values xx and yy. If they are both bool, the result will be True or it will be False. If we evaluate (5 and 3), the result will be either 5 or it will be 3. But which one? Python first evaluates the bool value of the left side of the and; that is, bool(xx). (The bool value of a non-zero number or a non-empty list or string is True. The bool value of zero or of an empty list or string is False.) If that value is False, we know right away that bool(xx and yy) will be False, so and returns the value xx (it could be False or 0 or an empty list, [], or an empty string "", etc.). But if bool(xx) is True, we don't know the result, as it depends on yy. So then and returns the value of yy. As a function, and would work like this :
    def andFunction(xx,yy): result = xx if bool(result): result = yy return result
  4. Finally, evaluate ors left to right. Like and, xx or yy returns either xx or it returns yy. Can you figure out why the following or function works? Once again remember that or will evaluate yy, the argument on the right, only if it has to.
    def orFunction(xx,yy): result = xx if not bool(result): result = yy return result

Putting all these rules together, why does the QuizMaster expression at the beginning of this section work? The expression is 1 == 1 and 2 or 3. There are no mathematical subexpressions, so we proceed to comparisons. 1 == 1 is True, so it becomes True and 2 or 3. The left side of the and is True, so and returns the right side, which is 2. We then have 2 or 3. The left side of or has a boolean value of True, so or returns the left side: 2. The answer that is printed is 2.

QUESTIONS ON RANGE EXPRESSIONS

What is printed by print range(5,3,-1) ? [5,4] Correct!

The range function returns a list of equally-spaced integer values. There are three variants, depending on whether you use 1, 2 or 3 integer arguments in your call. If there are no possible values, range will return an empty list, represented by [].

  1. 3-argument variant: range(start, stop, step). The parameter start provides the first value that will appear in the list, if any. Subsequent values will be start + step, start + 2*step, start + 3*step, etc. If step > 0, the sequence will be increasing; if step < 0, the sequence will be decreasing. If step is 0, range will generate a ValueError exception. The sequence always stops before reaching stop; the stop value will never appear in the list. The list will be empty in two cases: (a) start == stop, and (b) stop - start has the opposite sign from step.
  2. 2-argument variant: range(start, stop). This is the same as the 3-argument variant, with an assumed step of 1. The list returned will be empty unless start < stop.
  3. 1-argument variant: range(stop). This is the same as the 3-argument variant, with an assumed start of 0 and step of 1. The number of entries in the returned list is the same as stop. For example, range(5) gives [0, 1, 2, 3, 4], which has 5 entries.

[3, 4, 5, 6, 7] # Hint: step is 1; list does NOT include 8 Correct! What is printed by print (range(-2,-1)) ? [-2] # Hint: the step is 1; cannot include -1 Correct! What is printed by print (range(3,-4,-2)) ? [3, 1, -1, -3] # Hint: cannot pass the stop value Correct!

QUESTIONS ON SLICE EXPRESSIONS

What is printed by print ('alphabet'[1:3]) ? "lp" Correct!

The slice operator (here [1:3]) can be used to extract part of any sequence in Python, whether it is a string, a list, a tuple, and later a numpy array�in fact, anything that can be subscripted. Here are the rules:

  1. The type doesn't change. The result of the slice will be of the same type as the sequence you are slicing: a list yields a list result, a string yields a string result, and a tuple yields a tuple result.
  2. The numbers in the slice brackets define a range of subscripts. For example, if you take a slice "martians"[3:6], you will get characters in positions defined by range(3,6), namely positions [3, 4, 5], yielding "tia". It is possible to add another part to the slice to provide a step, like "martians"[3:0:-1] which gives "tra".
  3. Negative numbers have the sequence length added to them. Negative entries in the slice (except in the last step part) are first converted by adding the length of the sequence before a range is generated. So "martians"[3:-1:-1] is interpreted as "martians"[3:7:-1] which is "", and "martians"[-5:-2] is the same as "martians"[3:6] which is "tia".
